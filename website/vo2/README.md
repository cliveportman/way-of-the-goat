# VO2 Max Calculator

A browser-based VO2 max estimator built in Rust compiled to WebAssembly. Upload a GPX file from any GPS watch or app and get instant estimates — all processing happens locally in the browser, no data is uploaded anywhere.

## How it works

GPX files contain GPS coordinates, timestamps, elevation, and optionally heart rate data recorded by your device. The app uses three estimation methods:

- **ACSM + Heart Rate (Steady-State)** — Uses the ACSM running metabolic equation to estimate oxygen consumption at each GPS point, then applies the HR/HRmax linearity assumption to back-calculate VO2 max. Requires heart rate data. Confidence is driven by the number of qualifying steady-state samples and how consistent they are (coefficient of variation).
- **Jack Daniels — Best N-min Effort** — Finds your fastest sustained segment using a sliding time window and applies the Daniels/Gilbert formula. Confidence accounts for both effort duration and HR effort level (easy/Zone 2 runs will significantly underestimate your true VO2 max).
- **Firstbeat (HR–VO2 Regression)** — Fits a linear regression across all steady-state (HR, VO2) pairs and extrapolates to HRmax. Confidence is driven by the R² of the regression fit.

## Prerequisites

- [Rust](https://rustup.rs/)
- [wasm-pack](https://rustwasm.github.io/wasm-pack/installer/)

```bash
cargo install wasm-pack
```

## Building

Run from this directory:

```bash
wasm-pack build --target web
```

This compiles the Rust source in `src/` and outputs the WebAssembly binary and JavaScript bindings into `pkg/`. The `pkg/` directory is committed so the calculator can be served without a build step, but must be regenerated whenever `src/lib.rs` changes.

## Serving

Any static HTTP server works. From this directory:

```bash
python3 -m http.server 8080
```

Then open `http://localhost:8080`.

> **Note:** The calculator must be served over HTTP — opening `index.html` directly as a `file://` URL will not work because browsers block WebAssembly module imports on the file protocol.

## Inputs

| Field | Required | Notes |
|-------|----------|-------|
| GPX file | Yes | Supports Garmin, Polar, Apple Watch, Strava exports |
| Body weight (kg) | No | Used to display absolute VO2 capacity in L/min |
| Max heart rate (bpm) | No | Estimated from recorded data if blank. Use the Tanaka formula (208 − 0.7 × age) if unknown |

## Known limitations

- The ACSM formula is validated for running only — cycling or hiking GPX files will give unreliable results.
- Jack Daniels significantly underestimates VO2 max for easy or Zone 2 runs (the formula assumes race-like effort).
- HR-based methods depend on the accuracy of your HRmax input. The standard deviation of age-predicted HRmax formulas is ±10–12 bpm.
- GPS noise on short segments can distort speed and grade calculations.
